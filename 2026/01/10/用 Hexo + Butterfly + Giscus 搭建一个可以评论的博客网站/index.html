用 Hexo + Butterfly + Giscus 搭建一个可以评论的博客网站

发表于2026-01-10|更新于2026-01-11|技术

前言

最近搭建了一个个人博客网站，使用了 Hexo 静态博客框架，配合 Butterfly 主题和 Giscus 评论系统。整个搭建过程相对简单，而且效果很好。在这里记录一下搭建过程，希望能帮助到也想搭建个人博客的朋友。

为什么选择 Hexo + Butterfly + Giscus

  • Hexo：快速、简洁且高效的静态博客框架，基于 Node.js，支持 Markdown，生成静态文件速度快
  • Butterfly：现代化、优雅且功能丰富的 Hexo 主题，支持多种自定义选项，界面美观
  • Giscus：基于 GitHub Discussions 的评论系统，无需后端，支持 GitHub 账号登录，完全免费

准备工作

在开始之前，你需要准备：

  1. Node.js 环境：安装 Node.js（建议使用 LTS 版本）
  2. Git：安装 Git 用于版本控制和部署
  3. GitHub 账号：用于托管博客代码和配置 Giscus 评论系统
  4. Gitee 账号（可选）：如果你想部署到 Gitee Pages

注意事项

  1. 主题配置：建议在项目根目录创建 _config.butterfly.yml 来覆盖主题配置，而不是直接修改主题文件
  2. 图片资源：图片可以放在 source/img/ 目录下统一管理
  3. 部署配置：确保 Git 仓库配置正确，并且有推送权限
  4. Giscus 配置：仓库必须是公开的，并且启用了 Discussions 功能

总结

使用 Hexo + Butterfly + Giscus 搭建个人博客是一个非常不错的选择，既简单又美观。整个过程不需要后端服务器，完全基于静态文件，部署方便，而且评论系统也很完善。
