作为一个自托管用户，你常常需要去访问自己托管的网页应用。

这篇文章告诉你，什么是hosts，为什么你需要去修改hosts，什么是反代，如何配置反代。

## 什么是hosts

最初在英特网的前身ARPANET中，其成员SRI International手动维护并分享了一个名为HOSTS.TXT的文件，其中就包括主机名称和对应地址，它可以看作是当时互联网的主机通讯录。

1983年DNS系统开始开发，1984年得到了发展，在网络快速的发展过程中，DNS可以自动提供动态的主机名解析。

不过在现代操作系统中，hosts文件仍然是一个可以作为备用手段的名称解析机制。

Windows的hosts在`C:\Windows\System32\drivers\etc\hosts`，Mac以及Linux则在`/etc/hosts`。

## 为什么需要修改hosts

如果你没钱购买域名，又需要一个短名称，那么hosts可能是你的唯一选择，因为你不需要再使用一套十分繁琐的命名方式`192.168.1.101:26578`，转而使用一套优雅的方式`app.myname.io`，并且每一个网页应用的访问端口都在80（如果你配置了反代的话），不需要去记繁杂的端口号；如果你将来有了域名，则可以直接把hosts的内容复制过去。

hosts在每个系统的格式都是通用的，就像下面这样：

```
# 注释
127.0.0.1 localhost

# 示范，不同的名字可以绑定一个IP
192.168.1.101 app.myname.io
192.168.1.101 anotherapp.myname.io

# 如果你有公网IP的话（假的公网IP，示范用的）
142.142.142.142 app.myname.io
142.142.142.142 anotherapp.myname.io

# 但是不同的IP不能绑定一个名字（错误示范）
192.168.1.101 app.myname.io
192.168.1.102 app.myname.io
```

## 什么是反代

为什么不同的名字可以绑定一个IP实现应用跳转呢？这需要了解一个词，反代。

反向代理是一种位于客户端和服务器之间的中间服务器，主要用于隐藏后端服务器的真实地址，提升安全性和负载均衡能力。

我们假设一个服务器部署的服务有以下几个端口：

```
192.168.1.101:27856
192.168.1.101:6789
192.168.1.101:3000
```

当我们配置了反代，通常部署在80端口，那么当我们通过hosts里不同的名字访问80端口时，反代服务就会根据主机名自动选择转发流量到配置里对应服务的端口，以此实现每个服务的端口都可以在80，并且每个服务都可以拥有一个独立的域名：

```
# 统统都是80端口，日常可以直接省略
mymusic.myname.io:80
mynote.myname.io:80
mypaint.myname.io:80

# 不同的应用，全部指向一个地址
192.168.1.101:80
```
